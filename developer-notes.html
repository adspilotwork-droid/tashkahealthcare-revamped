SEO Change #1

Title Tags — All 4 Live Pages Fixed

All pages used an identical placeholder title from the HTML theme. Google was indexing the site as "Artificial Intelligence HTML Template".
P1 · CriticalFixed
What Was Wrong
All 4 live pages (homepage, /hospital_management, /custom_ai, /contact) had the exact same title: "Tashka Tech - Artificial Intelligence HTML Template". This is a placeholder from the HTML theme that was never updated. Google uses the title tag as its #1 on-page ranking signal.
SEO Benefit
Unique keyword-optimised titles tell Google exactly what each page is about. Title tags directly influence: (1) which keywords each page ranks for, (2) SERP click-through rate — a compelling title increases clicks, which signals quality to Google. Fixing duplicate titles is the single highest-impact technical SEO change possible.
Expected Impact
Immediate — Google re-crawls and re-indexes within 1–2 weeks of deployment. Pages will begin appearing for brand keyword searches within days. Organic impressions for target keywords should begin growing within 4–6 weeks.
⬆ High Impact

SEO Change #2

Meta Descriptions + Open Graph Tags — Added to All Pages

No meta descriptions existed. No Open Graph tags. Social sharing showed blank previews. SERP snippets were auto-generated from random body text.
P1Fixed

Zero meta descriptions on any live page. Zero Open Graph tags (og:title, og:description, og:image). Zero Twitter card tags. Google uses CTR as a quality signal — higher CTR pages get ranking boosts. OG tags enable rich social previews on LinkedIn, WhatsApp, Twitter — critical for B2B content sharing where decision-makers discover software.
Expected Impact
CTR improvement of 15–40% from better SERP snippets. Improved social sharing engagement. LinkedIn previews showing product name and description instead of blank preview. Effect visible within 2–3 weeks of Google re-indexing.

SEO Change #3

Structured Data (JSON-LD) — Added Across All Pages

Zero structured data on the live site. Google could not surface rich results. LLMs (ChatGPT, Gemini, Perplexity) had no entity signal to cite Tashka Healthcare in AI-generated answers.
P1Fixed
What Was Wrong
Not a single JSON-LD, Microdata, or RDFa tag on any live page. The /hospital_management page has 4 FAQs in its body content but no FAQPage schema — meaning Google cannot show them as expandable SERP accordions. No LocalBusiness schema = no rich local results. No SoftwareApplication schema = no software rich results.
SEO Benefit
Structured data enables: (1) FAQ accordions in SERPs — doubles SERP real estate for commercial queries, (2) Knowledge Panel eligibility for brand searches, (3) Local pack rich results for "hospital software Bangalore", (4) Software rich results with ratings, (5) LLM citations — ChatGPT and Gemini use Organization @id to cite specific businesses in AI answers about hospital software in India.
Expected Impact
FAQ schema → SERP accordion features within 4–8 weeks. LocalBusiness schema → local pack eligibility. Organization @id → brand Knowledge Panel within 8–16 weeks. LLM citation likelihood increases progressively as content is indexed and cited.
⬆ High Impact
Before — No structured data on any page

SEO Change #6

URL Slugs — Underscores Replaced with Hyphens

Live site uses underscore URLs (/custom_ai, /hospital_management). Google treats underscores as word connectors — "custom_ai" reads as one word "customai", not "custom ai".
P1Fixed
What Was Wrong
Google's John Mueller has explicitly stated underscores in URLs are treated as word connectors. /hospital_management = "hospitalmanagement" in Google's index. This prevents the URL from providing keyword relevance signals for "hospital management" as two separate words.
SEO Benefit
Hyphenated URLs: /hospital-management reads as "hospital" + "management" — two words that match keyword searches. Also applies to /clinical-ai (was /custom_ai — wrong name entirely, "custom ai" has near-zero search volume; "clinical AI" is the correct search term).
Expected Impact
Incremental ranking improvement from URL keyword signals. More important: 301 redirects must be set up on production server to preserve any existing backlinks. Without 301s, all current backlinks pointing to /custom_ai and /hospital_management become dead links.
↗ Medium Impact

SEO Change #8

NAP Consistency — Standardised Across All Pages and Schemas

Live site has two different addresses and two different phone numbers across its pages. NAP inconsistency is a confirmed local SEO ranking negative.
P1Fixed
What Was Wrong
Homepage footer: "No.14, Whitely Layout Subba Rao Compound... 560066" / +91 91081 98087. Contact page: "No 5, Whitely Layout, Whitefiled, Bengaluru - 560066" / +91 99728 60316. Consistent NAP across all pages, all schemas, and all citations (when submitted to G2, Clutch, etc.) builds a strong entity signal. Google's local algorithm associates the NAP with the Google Business Profile listing — consistency strengthens local pack ranking for "hospital software Bangalore" type searches.
Expected Impact
Foundational for local SEO. Effect compounds as more external citations are built with the canonical NAP. Local pack ranking improvement within 8–12 weeks after consistent citations are established.

301 Redirects — Required on Production Server

🚨
Critical: Set these up BEFORE removing the old pages. Without 301s, any existing backlinks to the old URLs become dead links and link equity is lost.
# Apache .htaccess

Redirect 301 /hospital_management /hospital-management
Redirect 301 /medical_coding /epr-software
