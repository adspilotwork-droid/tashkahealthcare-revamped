🔧 Technical SEO 44/100

The most urgent category. Multiple P1 issues were directly impacting Google's ability to crawl, index, and rank these pages. Most have been fixed in the preview site.

robots.txt not confirmed — may be blocking Googlebot
P1⚠ Unverified
Cannot confirm presence or content of /robots.txt. If absent, Googlebot still crawls but there is no Sitemap directive pointing to the XML sitemap. If misconfigured, critical pages could be blocked.
Create /robots.txt at domain root. Add: Sitemap: https://tashkahealthcare.com/sitemap.xml. Verify in Google Search Console → Settings → robots.txt.
XML Sitemap not submitted to Google Search Console
P1⚠ Unverified
No confirmation that sitemap.xml exists or has been submitted to GSC. Without a submitted sitemap, Google relies on crawl discovery alone — slower indexing, especially for new pages like /epr-software.
Create sitemap.xml listing all 5 pages. Submit via GSC → Sitemaps. Re-submit whenever a new page is published.
Google Search Console not confirmed as set up
P1Missing
No GSC property confirmed. Without GSC, there is no crawl error visibility, no index coverage data, no manual indexing requests, and no Core Web Vitals reporting. This is the single most important free SEO tool — not having it is flying blind.
Verify domain ownership in GSC via DNS TXT record. Submit sitemap. Use URL Inspection to request indexing for all 5 pages immediately after production deploy.
Google Analytics 4 not detected on any page
P1Missing
No tracking code found in page source across all pages audited. Without GA4, there is zero data on which pages drive demo requests, where users drop off, which keywords bring converting traffic, or what mobile vs desktop split looks like.
Create GA4 property. Add gtag.js snippet to <head> of all pages before production. Set up conversion events: form_submit, whatsapp_click, demo_click, phone_click.
Page speed — render-blocking scripts, no image optimization confirmed
P2⚠ Likely Fail
Core Web Vitals are a confirmed Google ranking signal since 2021. Render-blocking scripts present in source. No evidence of lazy loading or image compression. Google's PageSpeed Insights score not yet measured.
Run PageSpeed Insights on all pages. Defer non-critical JS. Compress images (use WebP format). Add loading="lazy" to below-fold images. Target LCP under 2.5s.
301 redirects needed for URL slug changes
P2Action Required
Preview site uses corrected hyphenated URLs (/clinical-ai, /hospital-management, /epr-software). When deployed to production, the old underscore URLs (/custom_ai, /hospital_management) will 404 unless redirected. Any existing backlinks to old URLs will be lost without 301s.
Set up server-side 301 redirects: /custom_ai → /clinical-ai and /hospital_management → /hospital-management. Update any existing internal links pointing to old URLs.
OG image file does not exist
P2Missing
All pages reference /img/og/tashka-og.jpg in og:image meta tags but this file does not currently exist. When pages are shared on LinkedIn, WhatsApp, or Twitter, no preview image will appear — significantly reducing click-through from social sharing.
Create a 1200×630px branded image per page (at minimum one shared image). Place at /img/og/tashka-og.jpg. Ideally create page-specific variants for EPR, Clinical AI, and Hospital Management pages.

✍️ Content & UX 61/100

The strongest content category — but with critical trust gaps. Reviews section has been added. The remaining gaps require input from the client.

Case studies have no quantified metrics — "reduced consultation time" is not a stat
P1High Impact
Chirag, Soukhyada, and CSI case studies are present but use vague language: "reduced consultation time", "improved patient throughput". Enterprise buyers need numbers. A stat like "OPD consult time reduced from 8 mins to 90 seconds" is worth 10x more for conversion and SEO than a generic claim.
Request specific before/after numbers from each hospital client. Even 2–3 concrete metrics per case study will dramatically improve both trust and content depth.
No named testimonials with attribution — reviews section uses representative quotes
P1
The reviews section in the preview site uses representative/illustrative quotes. For production, these must be replaced with actual verbatim reviews from the Google Business Profile — with real names and hospitals. Using fabricated quotes violates Google's review guidelines and reduces trust for sophisticated buyers.
Once the GBP has 5+ reviews, copy verbatim text and reviewer names from the GBP listing into the reviews section HTML. Update the review cards with actual data.
No About Us page — "Built by doctors" claim not substantiated
P2
"Built by doctors" is a strong trust signal — but it's only in the footer. No founding story, no team page, no clinical credentials visible anywhere. Enterprise hospital buyers (CMOs, CIOs) want to know who they're buying from. This also impacts E-E-A-T (Experience, Expertise, Authoritativeness, Trustworthiness) which Google uses as a quality signal.
Create /about-us.html with: founding story, team photos + bios with clinical credentials, company timeline, and press/awards if any. Add to navigation.
No blog or resource center — zero topical authority content
P2
Competitor Suvarna has multiple product pages, case studies, and brochure downloads. Tashka has no informational content targeting the research phase of the buyer journey. Keywords like "EPR vs HIMS explained", "NABH compliance checklist", "AI medical coding India guide" drive significant traffic and build topical authority for commercial queries.
Launch blog with 2 articles/month minimum. Priority topics: "EPR vs HIMS: What's the difference?", "NABH accreditation requirements for hospital software", "ICD-10 medical coding automation in Indian hospitals".
No /pricing page — missing pricing-intent search traffic
P3
"Hospital management software pricing India" and "EPR software cost" are high-intent queries. Buyers who search pricing are ready to evaluate — not research. Even a page that says "pricing upon discussion" with module tiers captures this intent.
Create /pricing.html with: module breakdown, size tiers (clinic / single hospital / multi-location), and "Get a Quote" CTA. No need to list exact prices.

🔗 Authority & Trust 30/100

The lowest-scoring category and the hardest to fix quickly — domain authority is built over months. But the following actions have the highest ROI for a new site.

Domain Authority near zero — no external citations or backlinks confirmed
P1
A new domain with no earned backlinks will struggle to rank for competitive keywords regardless of on-page quality. Authority is the primary differentiator between sites on page 1 vs page 3. Suvarna has been building citations for years.
Priority citation targets: Clutch.co (healthcare software category), G2.com (hospital management software), NASSCOM directory, Software Advice, Capterra. These are free listings with high domain authority that pass link equity.
Google Business Profile not claimed or optimized
P1
The GBP link (maps.app.goo.gl/jXKy1bp1iDax4QeU6) exists but the profile needs to be actively managed. GBP is the #1 local ranking factor for "hospital software Bangalore" type searches. Reviews, posts, photos, and Q&A all contribute to local pack visibility.
Claim and verify the GBP. Complete all sections: description (use primary keywords), services (EPR software, clinical AI, hospital management), photos (logo, team, office), opening hours. Post weekly updates. Actively request reviews from deployed hospital clients.
SSL/HTTPS confirmed ✓ — but no HSTS header verified
P2
HTTPS is correctly implemented (positive signal). However, HSTS (HTTP Strict Transport Security) header is not confirmed in server response headers. HSTS prevents SSL-stripping attacks and is a minor additional trust signal for enterprise security-conscious buyers.
Add HSTS header via server config: Strict-Transport-Security: max-age=31536000; includeSubDomains. Verify with securityheaders.com.
